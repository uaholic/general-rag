## 接口文档

### 1. 管理端页面接口

------

#### 1.1 登录页面

```text
GET /login
```

返回：

```text
登录页面 HTML
```

------

#### 1.2 提交登录

```text
POST /login
```

请求参数：

```json
{
  "secret": "123456"
}
```

成功：

```text
写入 cookie，跳转 /admin
```

失败：

```text
返回登录失败提示
```

------

#### 1.3 退出登录

```text
POST /logout
```

功能：

```text
清除登录 cookie，跳转 /login
```

------

#### 1.4 后台首页

```text
GET /admin
```

返回：

```text
后台首页 HTML
```

### 3. 业务线配置接口

------

#### 3.1 获取业务线配置页面

```text
GET /admin/business-line
```

返回：

```text
业务线配置页面 HTML
```

------

#### 3.2 新建 / 更新业务线配置

```text
POST /admin/business-line/save
```

请求参数：

```json
{
  "business_line_id": "business_line_course",
  "business_line_name": "大模型学习助手",
  "business_line_description": "用于回答大模型、RAG、Agent、LangChain、Milvus、Embedding 等学习资料相关问题。",
  "scenario": "课程答疑、项目文档问答、技术知识库问答",
  "target_user": "正在学习 AI 应用开发的学生和开发者",
  "assistant_role": "你是一个耐心、专业、能用通俗语言解释复杂技术概念的 AI 学习助手。",
  "welcome_message": "你好，我是课程知识库助手，可以帮你解答大模型学习问题。",
  "fallback_message": "抱歉，当前业务线绑定的知识库中没有找到明确依据。",
  "prompt_extra": "回答时尽量结合示例解释。",
  "kb_ids": ["kb_course", "kb_milvus"],
  "enabled": true
}
```

返回：

```json
{
  "success": true,
  "message": "保存成功"
}
```

说明：

```text
business_line_id 新建时可以由前端填写或由后端生成。
kb_ids 表示该业务线绑定的知识库 ID 列表，保存时同步写入绑定关系。
用户端检索时只使用当前 business_line_id 绑定且 enabled = true 的知识库。
```

------

#### 3.3 启用 / 禁用业务线

```text
POST /admin/business-line/{business_line_id}/toggle
```

请求参数：

```json
{
  "enabled": false
}
```

返回：

```json
{
  "success": true,
  "message": "更新成功"
}
```

------

#### 3.4 删除业务线

```text
POST /admin/business-line/{business_line_id}/delete
```

返回：

```json
{
  "success": true,
  "message": "删除成功"
}
```

说明：

```text
删除业务线时同步删除业务线和知识库的绑定关系。
不删除知识库、文档、Milvus chunk 和历史聊天记录。
```

------

#### 3.5 获取业务线嵌入代码

```text
GET /admin/business-line/{business_line_id}/embed-code
```

返回：

```json
{
  "business_line_id": "business_line_course",
  "business_line_name": "大模型学习助手",
  "bound_knowledge_bases": [
    {"kb_id": "kb_course", "name": "大模型课程库"},
    {"kb_id": "kb_milvus", "name": "Milvus资料库"}
  ],
  "embed_code": "<script src=\"http://localhost:8000/static/chat-widget.js\" data-company-id=\"default_company\" data-business-line-id=\"business_line_course\" data-theme-color=\"#1677ff\"></script>"
}
```

### 4. 知识库接口

------

#### 4.1 获取知识库页面

```text
GET /admin/kb
```

返回：

```text
知识库管理页面 HTML。页面列表需要展示每个知识库当前绑定的业务线。
```

------

#### 4.2 创建知识库

```text
POST /admin/kb/create
```

请求参数：

```json
{
  "name": "大模型课程知识库",
  "description": "保存 RAG、Milvus、LangChain、Embedding 等学习资料。",
  "enabled": true
}
```

返回：

```json
{
  "success": true,
  "kb_id": "kb_xxx",
  "message": "创建成功",
  "bound_business_lines": []
}
```

------

#### 4.3 更新知识库

```text
POST /admin/kb/{kb_id}/update
```

请求参数：

```json
{
  "name": "大模型课程知识库",
  "description": "更新后的描述",
  "enabled": true
}
```

返回：

```json
{
  "success": true,
  "message": "更新成功"
}
```

说明：

```text
知识库和业务线的绑定关系主要在业务线编辑页维护。
知识库编辑页可以只读展示当前被哪些业务线绑定。
```

------

#### 4.4 启用 / 禁用知识库

```text
POST /admin/kb/{kb_id}/toggle
```

请求参数：

```json
{
  "enabled": true
}
```

说明：

```text
禁用知识库后，该知识库不再参与任何业务线的用户端检索。
绑定关系保留，重新启用后可继续被已绑定业务线检索。
```

返回：

```json
{
  "success": true,
  "enabled": true
}
```

------

#### 4.5 删除知识库

```text
POST /admin/kb/{kb_id}/delete
```

功能：

```text
删除 MySQL knowledge_base 记录
删除该知识库下的 document 记录
删除该知识库下的 document_image 记录
删除 Milvus 中对应 kb_id 的 chunk
可选删除 MinIO 文件
MongoDB 聊天记录不删除
```

返回：

```json
{
  "success": true,
  "message": "删除成功"
}
```

------

### 5. 文档接口

------

#### 5.1 获取文档页面

```text
GET /admin/documents
```

返回：

```text
文档管理页面 HTML
```

------

#### 5.2 上传文档

```text
POST /admin/documents/upload
```

请求类型：

```text
multipart/form-data
```

请求参数：

```text
kb_id: 知识库 ID
file: 上传文件
```

返回：

```json
{
  "success": true,
  "doc_id": "doc_xxx",
  "parse_status": "pending",
  "message": "上传成功，开始解析"
}
```

------

#### 5.3 重新解析文档

```text
POST /admin/documents/{doc_id}/reparse
```

功能：

```text
清理旧 chunk
清理旧图片记录，可选
重置 document 状态为 pending
重新执行导入链路
```

返回：

```json
{
  "success": true,
  "doc_id": "doc_xxx",
  "parse_status": "pending",
  "message": "已重新提交解析任务"
}
```

------

#### 5.4 删除文档

```text
POST /admin/documents/{doc_id}/delete
```

功能：

```text
删除 MySQL document 记录
删除 MySQL document_image 记录
删除 Milvus 中 doc_id 对应的 chunk
可选删除 MinIO 原始文件和图片
更新 knowledge_base 的 doc_count 和 chunk_count
MongoDB 聊天记录不删除
```

返回：

```json
{
  "success": true,
  "message": "删除成功"
}
```

### 10. 用户端 API

------

#### 10.1 获取聊天框配置

```text
GET /api/widget/config
```

请求参数：

```text
company_id，可选
business_line_id，必填
theme_color，可选
```

返回：

```json
{
  "company_name": "XX智能科技有限公司",
  "business_line_id": "business_line_course",
  "business_line_name": "大模型学习助手",
  "welcome_message": "你好，我是课程知识库助手，可以帮你解答大模型学习问题。",
  "fallback_message": "抱歉，当前知识库中没有找到明确依据。",
  "bound_kb_ids": ["kb_course", "kb_milvus"],
  "theme_color": "#1677ff"
}
```

说明：

```text
如果 business_line_id 不存在或已禁用，返回 fallback 配置或错误提示。
bound_kb_ids 仅用于调试展示，前端不直接参与检索。
```

------

#### 10.2 流式聊天接口

```text
POST /api/chat/stream
```

请求参数：

```json
{
  "session_id": "session_xxx",
  "message": "RAG 的基本流程是什么？",
  "company_id": "default_company",
  "business_line_id": "business_line_course"
}
```

返回类型：

```text
text/event-stream
```

SSE 返回格式：

```text
event: rewrite
data: {"rewritten_query": "RAG 的基本流程是什么？"}

event: answer
data: {"delta": "RAG 的基本流程包括"}

event: answer
data: {"delta": "文档解析、向量检索和大模型生成回答。"}

event: references
data: {"references": [...]}

event: done
data: {}
```

------
