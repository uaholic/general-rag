```text
┌──────────────────────────────────────┐
│                                      │
│        轻量级企业知识库系统            │
```

## 字段


# 五、业务线配置页 `/admin/business-line`

## 页面目标

管理公司下的多个业务线。每个业务线拥有自己的聊天框嵌入代码，并绑定一个或多个知识库。

## 页面原型

```text
┌──────────────────────────────────────────────────────────────┐
│ 业务线配置                                      [ 新建业务线 ] │
│ 一个公司可以有多个业务线；每个业务线拥有自己的问答框。          │
├──────────────────────────────────────────────────────────────┤
│ 搜索：[ 搜索业务线名称、场景或知识库                         ] │
│      前端即时过滤当前列表，不触发后端请求                     │
│ │ 状态 │ 业务线           │ 场景         │ 绑定知识库  │ 操作 │ │
│ │ 启用 │ 大模型学习助手   │ 课程答疑     │ 课程库 Milvus│ 查看嵌入代码 编辑 删除 │
│ │ 启用 │ 售前咨询助手     │ 官网咨询     │ 产品库 价格库│ 查看嵌入代码 编辑 删除 │
│ │ 停用 │ 售后服务助手     │ 故障排查     │ 售后知识库   │ 查看嵌入代码 编辑 删除 │
└──────────────────────────────────────────────────────────────┘
```

## 操作

```text
新建业务线：
点击新建业务线，打开业务线编辑弹窗

编辑业务线：
点击编辑，打开业务线编辑弹窗

查看嵌入代码：
点击后打开当前业务线的嵌入代码弹窗

保存业务线：
POST /admin/business-line/save
```

## 校验

```text
业务线名称必填
回答角色建议必填
至少绑定一个知识库
```

# 六、知识库管理页 `/admin/kb`

## 页面目标

管理知识库。知识库可以被一个或多个业务线绑定；用户提问时只检索当前业务线绑定且启用的知识库。

## 页面原型

```text
┌──────────────────────────────────────────────────────────────┐
│ 知识库管理                                      [ 新建知识库 ] │
│ 用户提问时只检索当前业务线绑定且启用状态的知识库。             │
├──────────────────────────────────────────────────────────────┤
│ 搜索：┌──────────────────────────────────────────────┐        │
│      │ 输入知识库名称、描述或绑定业务线               │        │
│      │ 前端即时过滤当前列表，不触发后端请求           │        │
├──────────────────────────────────────────────────────────────┤
│ 知识库列表                                                    │
│ ┌────────┬────────────┬──────────┬────────────┬──────┬──────┬──────┐ │
│ │ 检索状态│ 知识库名称 │ 描述      │ 绑定业务线 │ 文档 │ Chunk│ 操作 │ │
│ ├────────┼────────────┼──────────┼────────────┼──────┼──────┼──────┤ │
│ │ 启用   │ 大模型课程库│ RAG资料   │ 大模型助手 │  5   │ 120  │ 编辑 删除 │
│ │ 启用   │ Milvus资料库│ 向量数据库│ 大模型助手 │  3   │  80  │ 编辑 删除 │
│ │ 停用   │ 价格政策库  │ 报价条款  │ 售前助手   │  2   │  41  │ 编辑 删除 │
│ └────────┴────────────┴──────────┴────────────┴──────┴──────┴──────┘ │
└──────────────────────────────────────────────────────────────┘
```

## 新建 / 编辑知识库弹窗

```text
┌──────────────────────────────┐
│ 新建知识库                    │
├──────────────────────────────┤
│ 知识库名称 *                  │
│ ┌──────────────────────────┐ │
│ │ 大模型课程知识库           │ │
│ └──────────────────────────┘ │
│                              │
│ 知识库描述                    │
│ ┌──────────────────────────┐ │
│ │ 保存 RAG、Milvus 等资料。 │ │
│ └──────────────────────────┘ │
│                              │
│ 当前被绑定业务线              │
│ ┌──────────────────────────┐ │
│ │ 大模型学习助手             │ │
│ └──────────────────────────┘ │
│                              │
│ 是否启用：[ 开关 ]            │
│                              │
│ [ 取消 ] [ 保存 ]             │
└──────────────────────────────┘
```

## 操作

```text
新建知识库：
POST /admin/kb/create

编辑知识库：
POST /admin/kb/{kb_id}/update

启用 / 禁用：
POST /admin/kb/{kb_id}/toggle

删除：
POST /admin/kb/{kb_id}/delete
```

## 删除确认

```text
删除知识库会同时删除该知识库下的文档记录和 Milvus chunk。
MongoDB 中历史聊天记录不会删除。
确认删除？
[取消] [确认删除]
```

# 十一、聊天记录列表页 `/admin/chat-sessions`

## 页面目标

查看用户会话列表，支持按 session、时间、关键词搜索，支持清空记录。

## 页面原型

```text
┌──────────────────────────────────────────────────────────────┐
│ 聊天记录                                         [ 清空全部 ] │
│ 查看用户通过聊天框产生的会话记录。                            │
├──────────────────────────────────────────────────────────────┤
│ 筛选区                                                        │
│ Session ID：[____________] 关键词：[____________] [搜索]       │
├──────────────────────────────────────────────────────────────┤
│ 会话列表                                                      │
│ ┌─────────────┬──────────────┬──────┬────────────┬────────┐ │
│ │ Session ID  │ 最后一条消息   │ 消息数│ 更新时间     │ 操作    │ │
│ ├─────────────┼──────────────┼──────┼────────────┼────────┤ │
│ │ session_001 │ RAG流程是什么 │  8   │ 2026-06-12 │ 查看 清空│ │
│ │ session_002 │ Milvus怎么用  │  4   │ 2026-06-12 │ 查看 清空│ │
│ └─────────────┴──────────────┴──────┴────────────┴────────┘ │
└──────────────────────────────────────────────────────────────┘
```

## 操作

```text
查看：
GET /admin/chat-sessions/{session_id}

清空单个会话：
POST /admin/chat-sessions/{session_id}/clear

清空全部：
POST /admin/chat-sessions/clear-all
```

## 清空确认

```text
确认清空该 session 的所有聊天记录？
该操作不会影响知识库和文档。
[取消] [确认清空]
```

# 十六、最终演示路径

```text
1. 打开 /login
2. 输入暗号进入后台
3. 进入企业信息页，配置公司信息
4. 进入业务线配置页，配置“大模型学习助手”并绑定知识库
5. 进入知识库管理页，创建“大模型课程知识库”
6. 进入文档管理页，上传带图片的 Markdown / PDF 学习文档
7. 等待文档解析成功
8. 进入问答测试页，测试 RAG 问答
9. 回到业务线配置页，点击“查看嵌入代码”，复制 JS 嵌入代码
10. 打开 demo.html
11. 右下角聊天框出现
12. 用户提问
13. 系统流式回答
14. 回答中展示图片
15. 回答下方展示引用来源
16. 返回后台聊天记录页，查看刚才的 session
```
